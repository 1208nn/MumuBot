## [Unreleased]

### 变更

- **协作约束现已补充核心架构边界**：`AGENTS.md` 新增话题主路径、学习系统职责、原始文本语义边界、“宁缺毋滥”质量取向与消息保留规则，后续协作默认按这些约束执行。
- **README 现已校准当前能力描述**：补充话题工作记忆、三档模型和管理后台说明，并修正 Milvus 依赖、后台访问等运行说明。
- **话题归属现已改为后台批量处理**：消息会先快速写入记录，再由后台按群攒批调用模型分配话题；低信息消息可以保持无话题，话题摘要也会异步刷新，减少聊天回复被话题整理阻塞的情况。
- **回复前分类现已同时服务语气和检索**：机器人每轮思考只做一次上下文分类，用同一结果选择发言风格并生成话题检索词；分类失败或检索词为空时会跳过归档话题和主动长期记忆检索。
- **学习系统现已改为只消费话题系统已处理消息**：learner 只学习黑话、风格卡片和全局成员画像，并按连续已处理前缀推进水位；未归属话题的 pending 消息会留给话题系统先处理，不再与长期记忆下沉链路重复扫原文。
- **关停后的待分配消息现已改为启动补偿**：关闭时不再把未分配消息伪造为无话题，保留真实 pending 状态；下次启动会扫描历史 pending 并强制补跑一次话题归属，即使未满批次也会处理。
- **仓库现已清理未引用实现与全部测试代码**：移除了仓库内已不再被主线调用的冗余函数，并删除现有 Go 测试文件，后续 `go test ./...` 将主要作为编译级检查使用。

### 修复

- **群聊提及现已按消息段显示昵称**：收到 at 群友的消息时，若 OneBot 消息段提供可读昵称，机器人上下文会显示昵称；无法识别昵称时保留 QQ 号，避免误指代其他群友。
- **话题分配结果现已可区分**：消息记录会通过话题编号和分配原因区分待处理、已归属、已处理但无话题或处理失败，避免队列满、模型失败等情况被误看作仍在等待自动处理。
- **归档话题摘要向量现已按最新摘要维护**：活跃话题被归档让位后会先移除旧向量并触发后台摘要刷新，摘要推进后再写入新的归档话题向量，减少历史话题召回使用过期摘要。
- **长期记忆向量现已跟随状态变化**：候选和归档记忆会移除向量，活跃、旧版兼容记忆才参与向量召回；向量结果不足时会用关键词检索补足，降低不可用记忆挤占召回名额的概率。
- **长期记忆去重现已避免重复调用模型**：同值判断只使用已有正文和本次提取出的值摘要做确定性比较，不再为了比较旧记忆额外触发一次模型调用。
- **话题分配原因现已保留模型原文**：模型给出的 `reason` 会直接持久化，只有缺失时才回落到稳定码，便于区分真实语义 `no_topic` 与队列满、异常等操作性 fallback。
- **消息清理现已保护已分配话题和 learner 未处理消息**：消息清理除了保留最新消息外，也会保留所有已归属话题消息和 learner 水位之后的消息，避免话题上下文或未学习内容被提前清掉。

## [1.3.0] - 2026-04-26

### 新增

- **话题工作记忆系统现已成为主路径**：机器人会持续识别群聊中的当前话题、维护话题摘要、记录未完事项，并在后续回复中带入更稳定的话题上下文。
- **管理后台现已支持话题归档与回看**：可以查看进行中和已归档话题，按群号、状态、关键词筛选，并进入详情查看摘要变化和关联消息。
- **机器人现已具备主动整理话题摘要的工具**：对话过程中可以主动整理当前话题，并把后续可复用的信息沉淀到话题与长期记忆中。
- **长期记忆现已引入声明提取与规范化流程**：新增事实、经历、偏好、约束、目标等归类，并记录来源、状态、证据数量和事实键，减少重复与低可信记忆。
- **已归档话题现可参与向量召回**：新增话题摘要向量集合，后续对话可以从历史话题中召回相关上下文。
- **管理后台记忆页现已支持归档、恢复和删除操作**：列表可以按状态、归类、来源和关键词筛选，便于清理过期或不可信记忆。
- **根目录现已新增发布更新日志**：`CHANGELOG.md` 将作为后续 GitHub Release 的发布说明来源。

### 变更

- **模型配置现已统一为高、中、低三档（破坏性变更）**：高档模型负责实时思考与回复，中档模型负责话题摘要、后台学习和记忆合并，轻量模型负责群风格判定；升级时需要把旧的 `llm`、`auxiliary_model`、`style_classification_model` 配置迁移到 `model_tiers.high`、`model_tiers.mid`、`model_tiers.low`。
- **模型 API Key 环境变量现已按三档模型命名（破坏性变更）**：配置项调整为 `MUMU_MODEL_HIGH_API_KEY`、`MUMU_MODEL_MID_API_KEY`、`MUMU_MODEL_LOW_API_KEY`，旧的 `MUMU_LLM_API_KEY`、`MUMU_AUX_LLM_API_KEY`、`MUMU_STYLE_CLASSIFICATION_API_KEY` 不再作为模型配置读取。
- **机器人回复提示词现已接入更多上下文来源**：话题记忆、相关长期记忆、跨群经历和最近发言者画像会共同参与回复构建，减少脱离上下文的回答。
- **长期记忆检索现已优先生效中的记忆**：旧版记录仍会兼容读取，但排序上优先使用已经确认有效的记忆。
- **管理后台记忆页现已展示更完整的判断信息**：状态、归类、来源和证据数会直接显示，便于判断记忆是否可信、是否需要归档。
- **发布流程现已改为读取根目录更新日志**：GitHub Release 会从 `CHANGELOG.md` 提取对应版本段落，缺少版本段落时会直接失败。
- **协作规范现已要求同步维护更新日志**：Agent 完成会进入仓库的改动时，需要同步记录面向使用者和维护者的更新内容。
- **更新日志格式现已统一为中文分组与加粗摘要式条目**：每条记录先给出加粗结论，再补充影响范围和具体说明，便于发布时直接阅读。

### 修复

- **管理后台状态筛选按钮现已可正常点击**：修复风格卡片和黑话列表中的状态筛选控件无法点击的问题。
- **话题写入异常现已不再中断收消息主流程**：话题工作记忆写入失败时会记录错误，但仍会保留运行时消息缓冲、继续成员画像更新与后续思考调度，避免临时存储问题导致机器人不响应。
- **主动长期记忆检索现已只使用话题模块生成的检索词**：话题上下文构建失败或未生成检索词时会跳过本轮主动召回，不再把聊天展示文本硬塞进长期记忆检索。
- **检索和学习文本现已避免使用消息渲染元信息**：话题匹配、归档话题召回、主动长期记忆检索、黑话匹配、风格分类、话题摘要和成员画像学习只使用原始文本；原始文本为空的图片、表情包或纯媒体消息会跳过这些语义处理，减少时间、消息 ID、昵称等渲染元信息对结果的干扰。
